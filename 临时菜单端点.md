# 临时解决方案：添加菜单导航端点

## 问题
VueCMF 前端登录后需要获取菜单数据（`/api/v1/router/menu/nav`），但该端点不存在。

## 快速修复方案

### 方案 1: 在 Central Brain 中添加临时菜单端点

在 `central-brain/main.go` 中添加：

```go
// 临时菜单导航端点
r.GET("/api/v1/router/menu/nav", func(c *gin.Context) {
    // 返回基础菜单结构
    menus := []map[string]interface{}{
        {
            "id": 1,
            "pid": 0,
            "model_id": 1,
            "title": "首页",
            "path": "/dashboard",
            "icon": "HomeFilled",
            "sort_num": 1,
            "status": 10,
        },
        {
            "id": 2,
            "pid": 0,
            "model_id": 2,
            "title": "系统管理",
            "path": "/system",
            "icon": "Setting",
            "sort_num": 2,
            "status": 10,
        },
    }
    
    c.JSON(200, gin.H{
        "code": 200,
        "msg": "success",
        "data": menus,
    })
})
```

### 方案 2: 在数据库中创建菜单表并返回数据

```sql
CREATE TABLE IF NOT EXISTS vuecmf_menu (
    menu_id SERIAL PRIMARY KEY,
    pid INTEGER DEFAULT 0,
    model_id INTEGER,
    title VARCHAR(50) NOT NULL,
    path VARCHAR(100),
    icon VARCHAR(50),
    sort_num INTEGER DEFAULT 0,
    status INTEGER DEFAULT 10,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO vuecmf_menu (pid, model_id, title, path, icon, sort_num) VALUES
(0, 1, '首页', '/dashboard', 'HomeFilled', 1),
(0, 2, '系统管理', '/system', 'Setting', 2),
(2, 3, '用户管理', '/system/users', 'User', 1),
(2, 4, '角色管理', '/system/roles', 'UserFilled', 2),
(2, 5, '权限管理', '/system/permissions', 'Lock', 3);
```
